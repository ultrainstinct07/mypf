## Overview

The **Phishing Detection System** is a high-performance security tool written in Python. It analyzes incoming URLs through multiple security filters to classify malicious phishing sites. By correlating URL heuristics, domain registration age, and threat intelligence feeds, the engine achieves a **95% classification accuracy** under production conditions.

---

## Detection Methodology & Layers

The detector applies three main analytical layers to evaluate the risk score of a given URL.

```
  [ URL Input ]
       │
       ├─► Layer 1: URL Heuristic Engine (Entropy & Subdomains)
       ├─► Layer 2: Domain Reputation (WHOIS Age & DNS Records)
       └─► Layer 3: Threat Intel (Correlation with Security APIs)
       │
       ▼ (Machine Learning Classifier)
  [ Risk Score Result (0-100) ]
```

### 1. URL Heuristic Analysis
Scans the lexical structure of the URL to detect spoofing patterns:
- **Subdomain Depth**: Checks for nested subdomain structures common in brand-spoofing campaigns (e.g., `paypal.com.verify-account.info`).
- **URL Entropy**: Calculates character entropy to detect obfuscated, random, or DGA (Domain Generation Algorithm) strings.
- **Lexical Keywords**: Flag patterns including `secure`, `login`, `bank`, `update`, and `verify`.

### 2. Domain & WHOIS Auditing
Performs real-time lookups to evaluate domain legitimacy:
- **WHOIS Lookup**: Queries domain registration age. Newly registered domains (< 30 days old) are immediately flagged with a higher risk multiplier.
- **DNS Verification**: Validates MX (mail exchange) records and name server configurations to ensure the domain is fully set up.

### 3. Threat Intelligence Feed Correlation
Aggregates known blacklists and reputation databases using public APIs:
- Automatically queries lists of active phishing targets (e.g., PhishTank, URLhaus).
- Integrates with security scanning APIs to retrieve historical domain ratings.

## Model Training & Classifier Performance
- **Dataset**: Trained on over 50,000 labeled URLs consisting of active phishing samples and safe benchmark URLs.
- **Model**: Employs an ensemble classifier (Random Forest + Gradient Boosting) evaluating 47 distinct features.
- **Accuracy**: Achieved **95% accuracy** on validation testing with a low false positive rate, ensuring practical reliability when integrated into local SOC workflows.
